Css units (em, rem, %) and responsiveness part - 1



Responsiveness has a dictionary defination of "The quality of reacting quickly and positively to certain condition".

Responsive web-design brings the same concept to websites and is defined asa way of building a website with the capablity to change their UI considerably to different screen sizes or device types. To build such a website css provides many ways and measunring units. Some of these units are em, rem and percentages (%). When we are
building responsive websites we do not use absolutely sized units like px, cm, mm and in, because the size of the element will not adapt and change to all the different screen sizes and device types unlike the relative units.

Css units (em, rem, %) and responsiveness part - 2



Why relative units



- When we are using % we are sizing an element relative to their parent and every time we access the site with different screen sizes it will always cover same percentage of the parent.



- well suited for designing scalable sections on a webpage that render both well on desktop and on mobile regardless of font-size in place.

- If we set our size in absolute unit (px, in, cm, mm) the end user will not have control over their browsing experince.

What happens if we change the default browser width?



Why relative units



The default width of the borwser is 16px and it is prefereable not to change it because some responsive websites could be affected with it. When relative units are used to design and develope a website specialy 'rem' it uses the root element's size which is the browser in this case to compute its size which is thought to be 16px in the time of development but if it is changed to some amount like 200px relative unit measures like 'rem' and 'em' behave unpredictably.
